# ADR-013: v0.9 版本规划调整 - 增加 CSV/Excel 导入功能

**状态**: ✅ 已接受
**日期**: 2026-02-05
**决策者**: hunkwk + AI
**影响范围**: v0.9 功能规划

---

## 📋 背景

### 原始 v0.9 规划

根据 ADR-012，v0.9 原计划为"用户优化版本"：

**原功能清单**:
1. CLI 用户体验优化（2 人日）
2. 可视化增强（3 人日）
3. 报告模板系统（2 人日）
4. 文档归档（1 人日）

**总工期**: 5-8 人日

### 用户需求

用户提出：将 CSV/Excel 导入功能从 v1.0 移至 v0.9。

---

## 💡 决策

### 调整方案

**新 v0.9 规划**: CSV/Excel 导入 + 用户体验优化

```
原 v0.9 规划:
├── CLI 用户体验优化（2 人日）
├── 可视化增强（3 人日）
├── 报告模板系统（2 人日）
└── 文档归档（1 人日）
总计: 8 人日

新 v0.9 规划:
├── CSV/Excel 数据导入（3 人日）- 新增 P0
├── 可视化增强（2 人日）- 保持 P1
├── CLI 基础优化（1 人日）- 压缩 P1
├── Markdown 报告模板（1 人日）- 简化 P2
└── 文档归档（0.5 人日）- 压缩 P2
总计: 7.5-8 人日
```

---

## 📊 决策依据

### 优势

1. **满足用户核心需求**
   - 数据导入是高频使用场景
   - Excel 是用户熟悉的工具
   - 降低技术门槛

2. **工期可控**
   - CSV/Excel 导入工作量评估准确（3 人日）
   - 总工期保持 7.5-8 人日（与原计划一致）

3. **技术验证**
   - v0.9 可以验证数据导入方案
   - 为 v1.0 Web UI 的数据导入功能做准备

---

## 🔄 替代方案

### 方案 A: 保持原计划（已否决）

**否决原因**: 数据导入是核心需求，不应推迟到 v1.0

### 方案 B: 数据导入 + 完整用户体验优化（已否决）

**否决原因**: 工期不可控，影响整体进度

### 方案 C: CSV/Excel 导入 + 核心体验优化（✅ 已接受）

**接受原因**: 平衡了功能需求和工期约束

---

## 📊 影响分析

### 工期影响

| 版本 | 原计划 | 新计划 | 变化 |
|------|--------|--------|------|
| v0.8.1 | 1-2 人日 | 1-2 人日 | 无变化 |
| v0.9 | 5-8 人日 | 7.5-8 人日 | +0-3 人日 |
| v1.0 | 20 人日 | 17-20 人日 | -0-3 人日 |

**说明**: v0.9 新增数据导入功能，但 v1.0 可以相应减少数据导入开发工作量，总体工期基本持平。

---

## ✅ 实施计划

### v0.9 功能优先级

```
P0（必须完成）:
└── CSV/Excel 数据导入（3 人日）

P1（重要）:
├── 可视化增强（2 人日）
└── CLI 基础优化（1 人日）

P2（可选）:
├── Markdown 报告模板（1 人日）
└── 文档归档（0.5 人日）
```
