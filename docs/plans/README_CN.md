# 计划目录

本目录包含版本化的实施计划。

## 目录结构

```
plans/
├── v0.1/              # 版本 0.1（当前）
│   ├── 001-docs-structure.md
│   └── summary.md
├── v0.2/              # 版本 0.2（计划中）
└── roadmap.md        # 版本路线图
```

## 版本命名

```
v{主版本}.{次版本}

主版本 (major): 重大架构变更、不兼容更新
次版本 (minor): 新增功能、向后兼容

示例:
  v0.1 - 初始版本
  v0.2 - 功能新增
  v1.0 - 稳定发布
```

## 计划文件

每个版本包含：
- **单独计划**: `NNN-{标题}.md`（编号）
- **总结**: 版本总体总结
- **状态**: pending（待办）| in-progress（进行中）| completed（已完成）

## 生命周期

1. 需求批准 → 在 `plans/vX.X/` 创建计划
2. 实施进行中 → 更新计划状态
3. 版本完成 → 归档并创建下一版本

## 集成关系

计划关联到：
- **需求**: `requirements/current/REQ-XXX.md`
- **活跃任务**: `active/{类型}-{功能}.md`
- **决策**: `decisions/NNN-{标题}.md`
